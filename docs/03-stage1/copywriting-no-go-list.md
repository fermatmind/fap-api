# 文案禁区清单（Copywriting No-Go List）— v0.3（对齐 API & 合规）

版本：**v0.3**  
适用范围：FAP（Fermat Assessment Platform）MBTI 主流程的所有用户可见文案：  
- 结果页（Profile / Report）  
- 分享卡（Share Asset）  
- 站内说明页（隐私/用户权益/免责声明）  
- 推送、活动页、公众号/社群投放素材  

目标：避免 **夸大疗效、误导诊断、恐吓式营销、隐私违规、歧视与攻击**，并与 v0.3 的 **字段/接口**（`content_package_version`、`scores_pct`、`axis_states`、`share_id`、`GET /attempts/{id}/share`）保持一致口径。

---

## 0. v0.3 新增字段/接口对文案的影响（必须遵守）

### 0.1 新增字段（文案可引用的“口径锚点”）

- `scores_pct`：五轴百分比（0–100）
  - 禁止把百分比解释成“比别人强多少”“智商/能力高低”“人格优劣”
- `axis_states`：五轴状态（very_weak/weak/moderate/clear/strong/very_strong）
  - 禁止把 `very_weak` 写成“你有问题/你不正常”
  - 必须使用“情境化/可变/更灵活”解释，不得恐吓
- `content_package_version`：内容包版本
  - 禁止在对外文案中暗示“版本越新越权威/越科学”
  - 版本仅用于内部追溯与回滚，不是营销话术
- `share_id`：分享追踪 ID
  - 禁止对用户展示为“可识别个人的ID”
  - 禁止把它当作“实名档案/心理档案编号”进行传播

### 0.2 新增接口（分享链路）

- `GET /api/v0.3/attempts/{attempt_id}/share`
  - 禁止文案暗示“我们会把你的作答明细分享出去”
  - 分享卡内容只允许使用该接口返回的模板字段（例如：`type_code/type_name/tagline/rarity/keywords/short_summary`）
  - 分享卡不得包含：手机号、邮箱、精确定位、设备标识、可回溯的测评明细

## 1. 高风险禁区（绝对禁止）

### 1.1 诊断/治疗暗示（医疗化表达）— 禁止

**禁止写法（示例）：**
- “你有抑郁症/焦虑症/人格障碍”
- “你需要立刻就医/你必须治疗”
- “本测试可以诊断心理疾病”
- “测完就能治好/改善失眠/治愈创伤”

**允许写法（替代）：**
- “这是偏好与倾向的描述，不是医学诊断”
- “如果你持续感到痛苦/影响生活，建议寻求专业帮助（心理咨询师/医生）”

### 1.2 绝对化、宿命论、贴死标签 — 禁止

**禁止写法：**
- “你就是这样的人，永远改不了”
- “你注定失败/注定不适合恋爱”
- “你必然离婚/必然被讨厌”
- “你一定会成功（或一定会失败）”

**允许写法：**
- “你更倾向于……在不同情境下可能呈现不同表现”
- “这是一种偏好，不代表固定命运”

### 1.3 恐吓式营销/焦虑营销 — 禁止

**禁止写法：**
- “不测就会错过人生机会”
- “你现在的性格很危险/会毁掉关系”
- “你再不改就来不及了”
- “你已经落后同龄人”

**允许写法：**
- “你可以把它当作自我观察的起点”
- “如果你愿意，我们提供一些可尝试的小建议”

### 1.4 歧视、攻击、羞辱、污名化 — 禁止

**禁止写法：**
- “内向就是懦弱/外向就是浮夸”
- “T 就冷血 / F 就玻璃心”
- “P 就拖延废物 / J 就控制狂”
- “T（敏感）就是矫情、情绪不稳定”

**允许写法：**
- 用中性描述替代价值评判：  
  - “更重逻辑/更重感受”  
  - “更偏计划/更偏弹性”  
  - “更易自我要求/更易保持稳定”

### 1.5 数据与隐私误导（合规底线）— 禁止

**禁止写法：**
- “我们不记录任何数据”（如果实际有 `events/attempts/results`）
- “分享卡不会被追踪”（如果你有 `share_id`/events）
- “你的作答会公开展示/会发给朋友”（除非用户主动分享）

**允许写法：**
- “我们会记录必要的数据用于生成报告与改进产品（详见用户权益/隐私说明）”
- “分享卡仅包含结果摘要，不包含作答明细”

## 2. 百分比与状态机口径禁区（v0.3 重点）

### 2.1 `scores_pct`（百分比）不得被解释为“能力高低/优劣”

**禁止写法：**
- “你比 90% 的人更强/更聪明/更优秀”
- “你外向能力 88 分，社交能力很强”
- “50% 说明你没主见/没性格”

**允许写法：**
- “百分比表示你在该轴两端之间的倾向位置与支持强度”
- “接近 50% 表示两端都能使用，情境切换更灵活”
- “更高表示倾向更稳定，但不代表优劣”

### 2.2 `axis_states`（very_weak 等）不得被污名化或恐吓

**禁止写法：**
- very_weak：  
  - “你很不正常/你有问题/你很矛盾所以不可靠”
- very_strong：  
  - “你绝对就是……（过度断言）”

**允许写法：**
- very_weak：  
  - “你在两端之间更灵活，可能会随场景变化”
- very_strong：  
  - “你的倾向更稳定，优势更集中，同时也更需要留意盲点”

### 2.3 “弱特质”必须使用“情境化”话术（模板）

当 `axis_states.* = very_weak` 时，允许使用以下结构（可复用）：

- “这条轴更接近中间，说明你在两端都有资源。”
- “你可能会根据场景选择不同策略：在熟悉环境更像 A，在陌生/压力情境更像 B。”
- “建议：当你不确定时，用一个小规则做选择（例如：先……再……）。”

## 3. 分享卡（Share Asset）专属禁区（对齐 `GET /attempts/{id}/share`）

### 3.1 分享卡禁止包含的信息

- 作答明细、题目内容、选项记录（任何形式）
- 可识别个人的信息：手机号、邮箱、身份证号、精确定位、家庭住址
- 可追溯的内部标识：数据库主键、完整 `attempt_id`、任何可直接查询个人记录的 ID  
  - **注意：`share_id` 仅内部使用，不对用户展示为“档案号”**

### 3.2 分享卡允许出现的信息（建议范围）

- `type_code` / `type_name`
- `tagline`（一句话签名）
- `rarity`（注意不要写成“稀有=更高级”）
- `keywords`（3–5 个中性标签）
- `short_summary`（1–2 句，不夸张、不诊断、不恐吓）
- 品牌标识与中性声明（可选一行）：
  - “仅供自我理解与交流，不构成诊断”

### 3.3 分享卡禁用话术（示例）

- “权威诊断报告”
- “人格缺陷/心理问题证明”
- “官方认证人格身份证（可用于求职/恋爱）”
- “测出你适合/不适合与某类人结婚”

## 4. 结果页/报告页禁区（Profile / Report）

### 4.1 不得把 MBTI 当作招聘/择偶硬筛工具

**禁止写法：**
- “XX 类型不适合当管理者/不适合谈恋爱”
- “找对象只找 XX 类型”
- “企业招聘建议淘汰 XX 类型”

**允许写法：**
- “在某些岗位/关系情境中可能更顺手，但仍取决于经验与环境”
- “建议把它作为沟通与协作的参考，而非筛选标准”

### 4.2 不得制造对立与引战

**禁止写法：**
- “T 比 F 更理性所以更优秀”
- “I 都社恐，E 都肤浅”
- “J 才靠谱，P 都拖延”

**允许写法：**
- “不同偏好各有优势：在不同任务与关系阶段会更有用”

## 5. 合规声明与免责声明（必须出现的位置）

### 5.1 必须出现（至少其一）

- 结果页底部（report/disclaimer）
- 分享卡角落（小字一行，可选）
- 隐私/用户权益页面（完整说明）

### 5.2 推荐的最短免责声明（可直接复用）

- “本测评用于自我理解与交流参考，不构成医学诊断或治疗建议。”
- “若你持续感到痛苦或功能受损，建议寻求专业帮助。”

## 6. 发布前文案自检（对齐 content-release-checklist v0.3）

每次发布内容包（`content_package_version`）前，必须做一次快速自检：

- [ ] 是否出现“诊断/治疗/疗效保证”等表达（1.1）
- [ ] 是否出现“绝对化/宿命论/贴死标签”（1.2）
- [ ] 是否出现“恐吓式营销”（1.3）
- [ ] 是否出现“歧视/羞辱/污名化”（1.4）
- [ ] 百分比/状态机解释是否正确（2.x）
- [ ] 分享卡是否包含任何敏感或可追溯个人信息（3.1）
- [ ] 是否把版本号当作营销“权威”话术（0.1）
- [ ] 是否在必要位置包含免责声明（5.x）

## 7. 典型“禁区→合规替换”对照表（速查）

| 禁区表达 | 风险 | 合规替换 |
|---|---|---|
| “你有抑郁症/焦虑症” | 医疗诊断 | “这是一种倾向描述，不是诊断；如影响生活请寻求专业帮助” |
| “你比别人强/更高级” | 价值评判 | “你更倾向于……不代表优劣” |
| “你注定失败/改不了” | 宿命论 | “你更常用某种策略；在不同场景也可能调整” |
| “very_weak=你有问题” | 污名化 | “更灵活/情境切换/两端都有资源” |
| “分享卡=档案号/认证身份证” | 隐私误导 | “分享卡是结果摘要，供交流参考” |
