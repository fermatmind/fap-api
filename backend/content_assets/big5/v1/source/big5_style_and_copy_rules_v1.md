# big5_style_and_copy_rules_v1

版本：v1
状态：冻结基线
用途：统一 Big Five 用户可见语言、标题、句式与 section heading 风格

---

## 1. 适用范围

本文件适用于以下所有 Big Five 用户可见内容：

- 正式长稿
- section/block 入库文案
- atomic / modifier / synergy / facet / action 输出
- shell / section heading / TOC / tools / lifecycle 文案
- compare / methodology / action matrix 用户可见 copy

不适用于：
- 内部 section key
- rule id
- registry technical labels
- scoring 字段名
- 研发调试文本

---

## 2. 标题与术语统一

### 2.1 section 标题统一表

| internal key | 用户可见标题 |
|---|---|
| `hero_summary` | 结果摘要 |
| `domains_overview` | 维度总览 |
| `domain_deep_dive` | 五维深解 |
| `facet_details` | 细分维度焦点 |
| `core_portrait` | 人格总览 |
| `norms_comparison` | 相对参照 |
| `action_plan` | 成长与下一步动作 |
| `methodology_and_access` | 方法与边界说明 |

规则：
1. section key 不改
2. 用户可见标题默认全中文
3. 不允许继续直接显示：
   - `Profile Summary`
   - `Domain Deep Dive`
   - `Norms Comparison`
   - `Methodology and Access`

### 2.2 高 / 中 / 低 的统一说法

内部资产或规则中允许：
- `H / M / L`
- `high / mid / low`

用户可见正文默认优先用：

- 较高
- 中位
- 较低

当需要更细分表达时，允许用：
- 偏高
- 中位偏高
- 中位偏低
- 偏低

不建议在正文里反复机械使用：
- HIGH / MID / LOW
- H / M / L
- low-mid / high-mid

### 2.3 “情绪性 / 神经质”命名规范

默认用户可见术语：
- **情绪性**

只在以下位置允许补充：
- 方法说明
- 研究/常模注释
- 专业括注

允许写法：
- 情绪性（对应 Big Five 文献中的 Neuroticism / 神经质）

禁止：
- 在整份中文结果页里反复混用“情绪性”和“神经质”
- 用“神经质”作为正文主要 section 标题

### 2.4 Compare 中文说法

用户可见 section 标题统一为：
- **相对参照**

正文用词优先：
- 参考样本
- 相对位置
- 百分位
- 相对突出
- 相对不突出

不建议：
- 继续用 `Norms Comparison` 作为主标题
- 用过多英文 statistical jargon 吓用户

### 2.5 Action Matrix bucket 中文说法

内部 bucket 保持：
- `continue`
- `start`
- `stop`
- `observe`

用户可见默认中文：
- `continue` → **继续放大**
- `start` → **开始尝试**
- `stop` → **停止减少**
- `observe` → **持续观察**

在空间有限的按钮/小标签中允许用短版：
- 继续
- 开始
- 停止
- 观察

## 3. 用户可见语言规则

### 3.1 推荐句式

优先使用以下句型：

- 你的系统倾向于……
- 你更容易……
- 这种结构的优势是……
- 这种结构的代价是……
- 这并不等于……而更像……
- 你真正需要处理的，不是……而是……
- 更准确的理解方式是……

### 3.2 禁止句式

以下句式默认禁用：

- 你就是……型的人
- 你注定会……
- 你永远都……
- 你天生不适合……
- 你最大的问题就是……
- 你太玻璃心了
- 你就是懒
- 你就是不成熟

### 3.3 反直觉发现写法模板

推荐格式：

- 你不是没尽责，你更像……
- 你不是社交差，你更像……
- 你不是脆，你更像……
- 你不是没有力量，而是力量更多来自……

### 3.4 解释顺序规则

所有正式正文与资产输出，优先遵循：

1. 先说明这维度 / 结构在测什么
2. 再说明你在什么位置
3. 再解释优势
4. 再解释代价
5. 最后再进入现实表现或建议

不要反过来一上来就给建议。

## 4. section heading 风格规则

### 4.1 标题风格

标题要求：
- 中文自然
- 4–8 字为主
- 不过度营销
- 不像功能菜单
- 不直译感太强

### 4.2 subtitle 风格

subtitle 只做一件事：
- 说明这一节为什么值得看

subtitle 不应该：
- 重复标题
- 只写空泛口号
- 直接沿用英文模板导语

### 4.3 英文暴露规则

用户可见正文默认不出现英文 heading。

允许英文出现的位置：
- 内部 key
- 技术说明
- 方法学括注
- 品牌固定名词（如 Big Five、FermatMind）

不允许英文出现的位置：
- section 标题
- section 副标题
- 正文导语
- compare/action 的主要说明句

## 5. 风格约束

### 5.1 整体风格

Big Five 正式 copy 必须满足：

- 专业
- 克制
- 有解释力
- 有现实感
- 不鸡汤
- 不命运论
- 不病理化
- 不过度类型化

### 5.2 允许的情绪强度

允许：
- 张力
- 压力
- 高负荷
- 警觉
- 内耗
- 消耗
- 恢复不足
- 进入门槛高

谨慎使用并需重写成结构解释：
- 焦虑
- 强势
- 冷
- 懒
- 脆弱
- 退缩
- 控制欲

### 5.3 不允许的两类偷懒写法

#### A. 类型化偷懒
- 你是天生的领导者
- 你天生就不适合合作
- 你注定只适合独处

#### B. 空泛疗愈偷懒
- 做最真实的自己就好
- 你很好，不需要改变
- 放下内耗，拥抱人生

Big Five 正式 copy 必须是：
**结构解释 + 现实表现 + 行动方向**
而不是：
**标签判断 + 情绪安抚**

## 6. 数值与百分位表达规则

### 6.1 百分位说法

推荐：
- 位于第 68 百分位
- 相对高于 68% 的参考样本
- 在参考样本中更突出
- 在这组比较对象里更明显

不推荐：
- 你比 68% 的人更焦虑
- 你赢过了 68% 的人
- 你属于高危人群（除非医学测量，Big Five 禁止）

### 6.2 不确定性与边界

每当正文涉及 compare / norms / relative position，必须提醒至少一次：

- 高低不等于好坏
- 相对位置不等于命运
- 这是倾向，不是宣判

---

## 7. Action copy 规范

### 7.1 Action 必须现实、具体、低摩擦

推荐：
- 本周可试的 4 个动作
- 先推进 20 分钟
- 先说一句边界句
- 连续两周记录触发点
- 每周回看一次

不推荐：
- 从今天开始彻底改变自己
- 每天都要坚持……
- 你必须立刻学会……
- 你应该永远……

### 7.2 Action 不得脱离 trait / scenario

所有建议必须能回到：
- 哪个 trait / structure 触发
- 哪个场景适用
- 为什么这个动作是最低摩擦杠杆

---

## 8. 重复与去重规则

### 8.1 hero 与正文去重
- shell hero 与 `hero_summary` 不可重复同一段 narrative
- 顶部工具区与正文 action 不可重复同一句 CTA
- `core_portrait` 不得简单重复五维总览

### 8.2 section 内去重
- `domain_deep_dive` 里不要把 broad trait 定义和 `domains_overview` 的短解释原封不动再写一遍
- `facet_details` 不要把 anomaly card 和 glossary row 写成一个意思
- `action_plan` 不要和 `methodology` 重复讲边界说明

---

## 9. 一句话收束

Big Five 正式 copy 的最终目标，不是让用户觉得“标签很准”，而是让用户觉得：

**“这份报告既解释了我为什么会这样，也告诉我现实里该怎么更好地用这套系统。”**
