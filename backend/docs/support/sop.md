# Report Lookup 售后 SOP（最小可执行）

版本：v0.1  
更新时间：2026-01-18

---

## 0. 目标
用最短路径帮助用户找回报告，并降低人工介入比例；同时确保合规与安全（不泄露报告）。

---

## 1. 客服决策树（按顺序问）

### Q1：你有“找回码 Ticket Code”或截图吗？
- 有 → 引导用户在任意端使用 Ticket Code 找回入口（或客服用内部工具查到 attempt_id 并提供“自助入口”）
- 没有 → 进入 Q2

### Q2：你还在原设备（同一手机/同一浏览器）吗？
- 是 → 引导用户打开“我的报告 / history”（Device Resume / 本地记录）
- 否 → 进入 Q3

### Q3：你愿意用手机号找回吗？
- 愿意 → 引导手机号登录/绑定（小程序一键取号 / Web OTP）
- 不愿意 → 进入 Q4

### Q4：你有订单号/支付截图吗（若涉及付费）？
- 有 → 走订单号找回（Order Lookup）或人工核验后提供“自助入口”
- 没有 → 进入 Q5

### Q5：你有邮箱吗？
- 有 → 走邮箱找回（MVP 可仅展示脱敏列表；正式版用 magic link）
- 没有 → 进入 Q6

### Q6：都没有（人工模糊查找）
收集字段：
- 大概测试时间段（日期/小时）
- 类型/结果大概是什么（可选）
- 若付费：金额、渠道、支付凭证截图
- 设备信息（机型/系统/微信版本）
结果：
- 若能定位 attempt → 提供临时访问链接或 encourage 手机号绑定
- 若不能定位 → 标记 unresolved，给出下一步建议（重新测试 + 保存票据码）

---

## 2. 红线（必须遵守）
1) 未核验身份不得发送完整报告内容（包括截图/长文复制）
2) 人工只发“临时访问链接”或引导自助找回，不直接转发报告内容
3) 不允许通过手机号/邮箱“明文枚举”查询（需要限频、盲回应策略）
4) 用户拒绝提供任何凭证时，只能建议重新测试并保存票据码

---

## 3. 常用话术（可复制）

### 票据码引导
“您这份报告有一个找回码（FMT-XXXX…），建议您在结果页复制或截图保存。若您有截图/找回码，我可以帮您快速定位。”

### 原设备召回
“如果您还在原来的手机上，一般在‘我的报告’里可以直接找到最近一次结果，我们先走这个最省事。”

### 手机号找回（推荐）
“为了确保跨设备也能找回，建议您用手机号登录/绑定。我们只用于登录、资产同步和售后通知，不会用于营销。”

### 没有任何凭证
“如果找回码、原设备、手机号/订单号都不方便提供，系统很难准确定位。建议您重新测评一次，并在结果页复制保存找回码，后续不会再丢。”

---

## 4. 工单字段（记录口径）

- Issue_Type：Lost_Asset
- Lookup_Method：Ticket / Device / Phone / Order / Email / Manual
- Resolution：Self_Served / Guided / Unresolved
- Root_Cause：No_Ticket_Saved / Cleared_Cache / Device_Change / Login_Mismatch / Unknown

建议附加字段：
- time_window（用户提供的时间段）
- payment_proof（有/无）
- risk_flags（枚举风险/异常频次）
