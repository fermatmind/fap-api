# 合规最小三件套（Compliance Basics · v0.2）

> 适用范围：Fermat Assessment Platform（费马测试平台）  
> 场景：MBTI 等「性格 / 兴趣类」测评相关产品与文案，不含医疗诊断与治疗。

---

## 0. 使用说明

本文件是「费马测试」现阶段的**合规底线与自检清单**，主要解决三个问题：

1. 我们在做的是什么类型的产品？（不是医疗）
2. 哪些话绝对不能说？（风险红线）
3. 日常要遵守哪些基础流程？（发布前要自查什么）

后续如果接入更重的量表（如抑郁筛查 / 关系评估等），本文件需要同步升级。

---

## 1. 产品定位与边界（我们是什么 / 不是什么）

### 1.1 当前产品定位

- 「费马测试」在 v0.2 阶段的主要定位是：
  - 面向 **普通大学生 / 社会青年** 的 **性格倾向 / 人格风格类测评工具**；
  - 提供 **自我认知参考** 与 **沟通辅助话题**；
  - 核心是「自我认识 + 人际沟通」，而不是「疾病诊断 + 治疗方案」。

### 1.2 明确不是的几类东西

所有页面（官网 / 小程序 / 分享图）都必须在文案设计与用户预期上清晰体现：

1. **不是医疗器械 / 医疗服务**
   - 不用于**诊断**任何精神疾病或心理疾病；
   - 不承诺任何**治疗效果**或症状改善。

2. **不是心理治疗 / 咨询本身**
   - 不替代面对面或在线的**专业心理咨询 / 精神科就诊**；
   - 不做一对一危机干预（如自杀干预）。

3. **不是法律、财务、学业等专业意见**
   - 不提供具备法律效力的意见或报告；
   - 不对用户的重大人生决策（退学 / 离职 / 离婚 / 投资等）给出“应该 / 必须”这类硬性指令。

### 1.3 对外表述建议

在合适的位置（官网测试介绍页、小程序测试说明、结果页底部）建议统一加入类似提示：

- 「本测试仅用于自我认知与沟通参考，不构成任何医学或心理诊断，也不能替代专业的心理咨询或医生就诊。」
- 「如你正经历持续且明显的痛苦（如长期失眠、情绪低落、有伤害自己或他人的想法），请及时联系专业心理咨询师或前往正规医院就诊。」

## 2. 高风险话术红线（绝对不能说的几类）

以下为当前阶段的**红线禁区**，任何正式对用户展示的内容（包括结果页、营销文案、分享图、客服 FAQ 等）都**不得出现或暗示**：

### 2.1 医疗与诊断相关

- 不得使用类似以下表达：
  - 「你有 XX 障碍 / XXX 症」
  - 「你的精神状态已经出现问题」
  - 「我们诊断你为……」
  - 「本测试结果表明你有严重心理疾病」
- 不得使用带有**治疗承诺**的表述：
  - 「通过我们的测试 / 服务，你可以治愈抑郁 / 焦虑」
  - 「按我们的建议做，就能痊愈」
  - 「保证治好你的心理问题」

**允许的安全表述示例：**

- 「你的回答显示，你在情绪敏感方面的倾向略高于平均水平，这可能意味着你对环境变化更敏感一些。」
- 「如果这些感受已经持续影响你的学习、睡眠或人际关系，建议及时与专业心理咨询师或医生沟通。」

### 2.2 自杀、自伤、暴力等极端行为鼓励

任何情况下都不得出现：

- 鼓励、赞美或浪漫化自杀、自残、自伤行为；
- 鼓励对他人暴力、报复、攻击；
- 暗示伤害自己 / 他人是解决问题的办法。

一旦在用户反馈或开放式文本中发现类似内容，应按**紧急流程**处理（见第 5 章）。

### 2.3 歧视、羞辱与极端标签化

不得出现针对以下任何群体的贬低、攻击或嘲笑：

- 性别 / 性取向
- 种族 / 国籍 / 地区
- 职业 / 收入 / 学历
- 宗教信仰 / 政治立场等

在人格类型描述中，避免以下倾向：

- 直接说「这种人格就是自私 / 懒惰 / 没出息」等；
- 把人格类型和价值高低直接挂钩（如「XX 型是最聪明 / 最高级人格」）。

**推荐做法：**

- 强调「倾向 / 风格」，用中性甚至略带温柔的表述，如：
  - 「你在做决定时更习惯先顾及他人的感受，这有助于维护关系，但有时也可能让你忽略自己的需求。」
- 即便提到可能的风险点，也建议加上可操作的建议，而不是一句“你不好”。

### 2.4 绝对化的命运论 / 结果绑定

避免把人格结果描述为：

- 「你天生就注定……」
- 「你永远都无法做到……」
- 「你这辈子都不适合……」

更推荐的表达方式：

- 「在人格倾向上，你*当前*更习惯……」
- 「这类倾向并不是绝对的，环境、经验和刻意练习都会带来改变。」

## 3. 用户数据与隐私（Data & Privacy）

> 现阶段平台以匿名 ID 为主，仍需要遵守基本数据安全原则。

### 3.1 数据最小化原则

- 不强制采集真实姓名、身份证号、详细住址等高敏信息；
- 若不需要手机号 / 邮箱完成当前功能，尽量不要求填写；
- 匿名标识建议采用：
  - 小程序 openid 的内部映射
  - 或本地生成的随机 UUID（如 `anon_id`）

### 3.2 存储与传输

- 确保传输使用 HTTPS（尤其是正式线上环境），避免明文传输；
- 在数据库中，严格区分：
  - **测评结果**（attempts / results）
  - **运营管理账号**（后台登录）
- 对于内部人员访问 DB 的权限，应控制在「必要最小范围」：
  - 谁能查原始答题？
  - 谁能查结果与事件日志？

### 3.3 对外展示与分享

- 结果分享图中，避免：
  - 拼接真实姓名 / 电话 / 具体地点；
  - 展示过细的隐私信息（如「你最近失恋」这类高度个人化且未经用户明确授权的信息）。
- 建议在分享图落款、小程序介绍中加入：
  - 「不展示用户具体作答内容，仅展示类型与部分概括性描述。」

## 4. 日常内容审核流程（Content Review Flow）

> 目标：任何会大规模被用户看到的内容，至少经过一次「合规视角」的自检。

### 4.1 适用范围

所有下列内容上线前，建议使用本流程自检一次：

- 官网文章 / 测试介绍页；
- 小程序题目文案 / 结果页文案；
- 分享图、广告文案、宣传海报；
- 公众号推文中关于测试的介绍语。

### 4.2 最小审核步骤

1. **作者自检**
   - 对照本文件（`compliance-basics.md`）和《copywriting-no-go-list.md》；
   - 特别关注：
     - 是否使用了「诊断 / 治疗」字眼？
     - 是否有把性格类型与价值高低绑定？
     - 是否涉及自伤 / 仇恨 / 歧视等内容？

2. **第二人 review（推荐）**
   - 若条件允许，让另一人以「小白用户 + 敏感用户」视角过一遍：
     - 有无让人产生被冒犯或被攻击的感觉？
     - 有无让人误以为这是医疗诊断的表述？

3. **留痕记录**
   - 在 Git 提交信息或 Notion / 飞书中简单记录：
     - 「已按合规清单自检」；
     - 如有争议内容，记录为什么认为可接受。

## 5. 紧急情况与用户求助（Emergency & Help）

> 现阶段你没有能力提供真正的危机干预服务，但至少要做到不误导、不抛弃用户。

### 5.1 高风险信号识别（文本层面）

如果在用户反馈 / 留言 / 客服对话中出现如下内容，应视为**潜在高风险**：

- 明确提到：
  - 「我想结束自己的生命」
  - 「我已经不想活了」
  - 「我有伤害自己 / 伤害某人的计划」
- 描述严重和持续性的痛苦：
  - 「每天都睡不着，只想哭」
  - 「完全无法上学 / 工作」

### 5.2 建议的安全回应原则（如有人工客服）

- 不做诊断、不下结论；
- 不给具体执行方案（例如改变用药等）；
- 建议的回应方向：
  - 表达理解和关切；
  - 鼓励对方寻求 **线下专业帮助**；
  - 如对方在中国大陆，可建议：
    - 就近前往有资质的精神卫生中心或综合医院心理科；
    - 联系身边信任的家人 / 朋友。

### 5.3 产品文案层面的提示

在涉及「负面情绪 / 压力」的结果页中，建议统一加入类似提示：

- 「本测试无法替代专业的心理评估或治疗，如果这些困扰已经严重影响你的生活，建议尽快向专业人士求助。」
- 「如你有强烈的自责、自我伤害想法，请尽快告诉信任的家人或朋友，并寻求专业帮助。」

## 6. 持续更新与迭代

### 6.1 版本管理

- 本文件建议放在仓库：`docs/compliance-basics.md`
- 每次重要修改，需在文件顶部或尾部记录版本与日期：
  - 如：`更新记录：2025-12-15 v0.2 首版`

### 6.2 与其他文档的关系

- 本文件与以下文档联动：
  - 《fap-v0.2-glossary.md》：统一术语与字段解释；
  - 《mbti-content-schema.md》：规范结果文案结构；
  - 《copywriting-no-go-list.md》：列出更细致的禁用话术。

### 6.3 何时需要升级本文件？

- 新增了「临床 / 半临床」性质的量表（如抑郁、焦虑、创伤等；
- 引入了付费咨询、导师一对一服务；
- 融入了第三方机构（比如医院 / 心理机构）的服务接口。

一旦出现上述情况，需要：

1. 与专业人士（心理咨询师 / 精神科医生 / 法律顾问）沟通；
2. 对本文件进行升级，增加更细致的风险分级、话术规范与应对流程。

---

> 总结：  
> 现阶段，把自己稳稳定位在「**人格 / 性格倾向类的自我认知工具**」，坚持不碰 **诊断 / 治疗 / 极端行为鼓励 / 歧视与仇恨** 四条红线，再配合基本的数据安全与审核流程，就能在「安全边界内」快速迭代产品。
