# AI 内容工作流质量门禁（Quality Gates）

本文件定义 AI 内容工作流的固定质量门禁。脚本化检查后置到闭环 10C，本闭环仅定义规则与人工检查流程。

## 通用门禁（所有类型）

- 禁词：绝对化、宿命论、道德评判、诊断/治疗暗示
  - 示例（非穷举）："必然" "注定" "唯一" "天生" "病态" "治愈"
- 结构：输出必须为 JSON；字段与任务包 8 schema/目录一致
- 风格：客观可解释、少 AI 味，不使用夸张或情绪化措辞
- 事实声明：
  - 统计/比例/分布/排名/对比必须来自 norms/whitepaper（任务包 6/9）并写明口径
  - 无可靠来源的事实性内容必须标注“推断/一般经验”

## 结构门禁（强制）

- role_card：标题 + 3–6 要点 + 3 行动建议 + 免责声明
- strategy_card：标题 + 场景/触发 + 3–5 策略 + 1 分钟行动 + 免责声明
- reads：标题 + 导语 + 正文 ≥ 3 段 + 小结 + 行动建议 + 免责声明

## 长度门禁（中文字符口径）

- 口径：按 UTF-8 字符计数，统计 CJK 字符与全角标点；不计空格/换行/英文字符
- role_card：500–900 字
- strategy_card：300–600 字
- reads：600–1200 字

## 重复率门禁

- 4-gram 重复率 > 8% 标红（默认阈值）
- 可选：根据品类分布调整阈值（需记录原因与版本）

## 执行流程

- 以人工 checklist 为主（见下）
- 脚本化检查后置到闭环 10C（本闭环不做脚本）

### 人工 checklist（最小集）

- 字段结构完整、无缺失
- 结构门禁符合（标题/要点/行动/免责声明）
- 长度落在区间内（中文字符口径）
- 禁词未出现、无诊断/治疗暗示
- 事实性声明已注明来源与口径；否则标注“推断/一般经验”
